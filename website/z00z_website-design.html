External Architecture Gates

NEAR pattern

Use concept-first documentation hubs plus explicit explorer access, data-infrastructure choices, and chain-abstraction examples that hide complexity without hiding risk.

Sui pattern

Separate onboarding, agent-ready skills, migration paths from other chains, onchain finance flows, stack primitives, and references so builders move from learning to exact APIs quickly.

Hyperliquid pattern

Keep a crisp product/protocol overview, builder tools, non-endorsement disclaimers, support categories, anti-scam warnings, risks, audits, developer API, and operational troubleshooting as distinct surfaces.

Evidence And Source Notes

The sitemap intentionally separates public education, protocol docs, developer docs, operator docs, security, research, support, and legal surfaces. Archive and planning material should be linked only through research/archive routes unless active docs or code promote it.
