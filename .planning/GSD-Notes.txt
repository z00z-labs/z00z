Контекст цели: нужен testnet с самыми важными, демонстрационными и инвестиционно-интересными features Z00Z:

* website / landing / docs portal
* wallet
* aggregators
* validators
* watchers
* DA / Celestia integration
* cross-chain features
* treasury / incentives
* practical use cases / demos
* end-to-end testnet flow

Что нужно сделать:

1. **Прочитать корпус документов**

   * внимательно просканируй все whitepaper/spec/architecture документы;
   * извлеки все идеи, модули, протоколы, use cases, криптографические требования, network components, off-chain/on-chain flows;
   * сгруппируй их по смыслу;
   * не теряй мелкие задачи: нужна mini-level task resolution, то есть задачи достаточно мелкие, чтобы их можно было реально назначать разработчикам.

2. **Проверить текущую кодовую базу**

   * определи, что уже реализовано;
   * что частично реализовано;
   * что отсутствует;
   * какие existing modules/files можно расширять;
   * где нужны новые модули;
   * какие части roadmap должны встроиться в уже существующую архитектуру, а не быть написаны отдельно;
   * отдельно отметь dead code, mock-only code, PoC-only code и места, требующие refactor.

   Разбей implementation plan на фазы:

   * Phase 0:
   * Phase 1:
   * Phase 2:
...
   * Phase N etc

   Каждой задаче присвой priority:

   * P1 — must-have для investor-grade testnet
   * P2 — strongly desirable, усиливает уникальность Z00Z
   * P3 — optional / later / nice-to-have

5. **Зависимости и параллельность**
   Для каждой задачи укажи:

   * depends_on
   * can_run_parallel_with
   * owner_role
   * estimated_effort
   * risk_level
   * definition_of_done

6. **Команда**
   Планируй под 5 разработчиков с активным использованием LLM/Codex:

   * Dev 1: Rust protocol/core
   * Dev 2: wallet/client
   * Dev 3: validator/aggregator/watchers
   * Dev 4: network
   * Dev 5: DA/Celestia/cross-chain
   * Dev 6: frontend/website/docs/usecase demos
     Если видишь лучшую разбивку — предложи её.

7. **Оценка сроков**
   Оцени, сколько времени займёт реализация:

   * optimistic
   * realistic
   * conservative
     Учитывай, что это не PoC, а testnet, который должен выглядеть убедительно для инвесторов.

8. **Mermaid Gantt**
   Сгенерируй полноценный `mermaid gantt`, где видно:

   * фазы;
   * последовательные задачи;
   * параллельные задачи;
   * критический путь;
   * задачи по website, wallet, aggregators, validators, watchers, cross-chain, treasury, Celestia DA, usecases;
   * длительность задач;
   * dependencies через `after`.

9. **Финальный output**
   Верни результат в такой структуре:

   * Executive summary
   * What already exists in codebase
   * Missing critical pieces
   * Feature groups
   * Phase roadmap
   * Detailed task table
   * 5-developer allocation
   * Mermaid Gantt
   * Timeline estimate
   * Critical risks
   * What can be shown to investors
   * What should be postponed after testnet

Важные правила:

* Не фантазируй: если в документах или коде чего-то нет — пометь как `not found / assumption`.
