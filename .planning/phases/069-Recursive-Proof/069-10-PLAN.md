# 069-10 Bounded Prover Lifecycle And Atomic Recovery

<objective>
Add a bounded, cancellable, observable proving lifecycle and atomic logical
multi-artifact persistence/recovery protocol so lag, timeout, memory/queue
exhaustion, restart, reorg, or parameter rotation affects shadow evidence only
and stale/partial artifacts never appear complete.
</objective>

## outputs

  rollup-owned Plan 09 integration seam. Jobs may use PP/PK only inside the
  prover boundary; verifier/watcher/wallet flows receive at most the cached
  generation-pinned VK bundle or proof-envelope digest.

## dependencies

- `069-09` receipt/store integration.

## acceptance_tests

- Queue, concurrent jobs, witness/memory/time budgets enforce typed failure without blocking canonical admission.
- Transient prover-only witness and assignment buffers minimize copies and are
  wiped through approved zeroize-on-drop wrappers on every exit path; retained
  archive material is not silently deleted.
- Crash at every proof/receipt/sidecar/manifest/anchor write boundary recovers idempotently and incomplete epochs remain invisible.
- Reorg/parameter rotation supersedes stale jobs/artifacts.

## simulation_gate

- Wall clock, fault scheduler, and remote worker boundary may be simulated; scheduler state, real proving/verifying, storage journal/commit/recovery, and canonical admission are live.

## negative_tests

- Queue overflow, timeout, memory/witness cap, cancellation race, stale fork/predecessor, parameter rotation, crash at each boundary, corrupted journal/commit, duplicate submission, witness/log secret leakage, transient-buffer cleanup gaps, CPU work on Tokio I/O executor.
